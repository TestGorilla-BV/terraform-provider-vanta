# Terraform Provider for Vanta

Manage [Vanta](https://www.vanta.com) resources through its v1 REST API. Built
on the [Terraform Plugin Framework](https://developer.hashicorp.com/terraform/plugin/framework),
this provider mirrors the architecture of `terraform-provider-holistics`.

## Authentication

Vanta authenticates with an **OAuth 2.0 client-credentials** grant. Create an
API client in Vanta (Settings → API tokens), then supply its credentials:

```hcl
provider "vanta" {
  client_id     = var.vanta_client_id
  client_secret = var.vanta_client_secret
  region        = "us" # "us" (commercial) or "gov" (FedRAMP); default "us"
}
```

The provider exchanges the credentials for a short-lived bearer token at the
region's token URL and refreshes it transparently. All settings can come from
the environment instead:

| Attribute       | Environment variable   | Notes |
|-----------------|------------------------|-------|
| `client_id`     | `VANTA_CLIENT_ID`      | Required unless `token` is set. |
| `client_secret` | `VANTA_CLIENT_SECRET`  | Required unless `token` is set. Sensitive. |
| `scope`         | `VANTA_SCOPE`          | Defaults to `vanta-api.all:read vanta-api.all:write`. |
| `token`         | `VANTA_API_TOKEN`      | Pre-obtained bearer token; skips the OAuth exchange. Sensitive. |
| `region`        | `VANTA_REGION`         | `us` or `gov`. Default `us`. |
| `base_url`      | `VANTA_BASE_URL`       | Override the API base URL (testing/proxies). |
| `token_url`     | `VANTA_TOKEN_URL`      | Override the OAuth token URL. |

## Supported resources & data sources

**Resources**

- `vanta_vendor` — a managed third-party vendor (full CRUD).
- `vanta_vendor_finding` — a finding attached to a vendor (full CRUD, nested
  under a vendor; composite import ID `"<vendor_id>/<finding_id>"`).

**Data sources**

- `vanta_vendors` — list vendors, filterable by name and status.
- `vanta_people` — list people, filterable by employment status.
- `vanta_frameworks` — list compliance frameworks with progress counters.
- `vanta_tests` — list tests, filterable by status / framework / integration.

The Vanta API is large; this provider implements a representative, production-
shaped slice. Adding more coverage is mechanical and needs no plumbing changes:

1. Add a typed client method in `internal/client/<resource>.go` (use the
   `paginate` helper for list endpoints).
2. Add a `*_resource.go` or `*_data_source.go` under `internal/provider/` and
   register its constructor in `provider.go`.
3. Add mock-server handlers and an acceptance test.

## Architecture

```
main.go                       # provider server entry point
internal/
  client/                     # HTTP client: OAuth token, Do(), pagination, errors
    client.go
    vendor.go vendor_finding.go people.go framework.go test.go
  provider/                   # provider, resources, data sources, helpers
    provider.go helpers.go
    vendor_resource.go vendor_finding_resource.go
    *_data_source.go
    *_test.go
  mockserver/                 # in-process API (incl. /oauth/token) for tests
examples/                     # HCL examples surfaced in the registry docs
docs/                         # generated by tfplugindocs
```

Key conventions carried over from the Holistics provider:

- A single `client.Do()` handles auth, JSON, 429 retries, and error parsing.
- `APIError` + `client.IsNotFound()` drive `RemoveResource` on 404 reads.
- `UseStateForUnknown` plan modifiers on every Computed / Optional+Computed
  attribute prevent spurious "(known after apply)" diffs.
- List endpoints are walked to completion via the generic `paginate` helper.

### A note on optional attributes

PATCH updates only send attributes that are set in configuration, so **removing
an optional attribute from configuration does not clear it server-side** —
set it to a new value to overwrite. This matches the underlying API's
merge-on-PATCH semantics.

## Development

```sh
make build        # compile the provider binary
make test         # unit tests
make testacc      # acceptance tests against the in-process mock (no creds needed)
make fmt vet      # format + static checks
make docs         # regenerate docs/ via tfplugindocs
```

Acceptance tests spin up `internal/mockserver` — including a mock
`/oauth/token` endpoint — so the full OAuth path is exercised without real
credentials.

## Releasing

Tagging `vX.Y.Z` triggers the CircleCI `release` workflow, which runs
GoReleaser to build multi-platform binaries, generate checksums, GPG-sign them,
and publish a GitHub release consumable by the Terraform Registry.
